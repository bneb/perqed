# Formalization Protocol

This protocol is a mandatory workflow for all formal verification efforts in the Perqed repository (Lean 4 or otherwise). It is designed to prevent the **"Formalization Gap"**—the false sense of security generated by having "zero sorrys" spread across disconnected lemmas that fail to logically bridge to the actual mathematical claim.

## The Prime Directive: Top-Down Signature First

Before writing any implementation code or helper lemmas, you **must** write the exact Lean signature of the target theorem as it appears in the English/mathematical claim.

### Rule 1: Literal Translation
The top-level theorem signature must be a literal translation of the target mathematical claim.
- If the claim involves a `limsup`, the formal signature must use `limsup`.
- If the claim involves a subsequence, the formal signature must quantify over subsequences.
- **Do not** substitute a global property (e.g., `∀ k, IsGreedy`) for an eventual or subsequence property.

### Rule 2: The `sorry` Anchor
Once the top-level signature is written, immediately append `sorry`.
```lean
theorem final_resolution (a : ℕ → ℕ)
    (h : Erdos265Sequence a)
    (hDual : DualRational a) : 
    limsup (fun n => (a n : ℝ)^(1 / 2^n)) atTop ≤ 1 := by
  sorry
```
This is the **Anchor**. All future work must aim exclusively at closing this specific `sorry`.

### Rule 3: No Goalpost Moving
You are strictly forbidden from altering the hypotheses of the Anchor theorem to make it easier to prove. 
- If you cannot close the `sorry` without adding a new hypothesis (e.g., `(hGreedy : IsGreedy a)`), then you have failed to formalize the theorem.
- If a helper lemma (`Island A`) cannot be formally composed with another helper lemma (`Island B`) to close the Anchor theorem, the English bridge between them is flawed, and the formalization is incomplete.

## Avoiding Islands of Truth
The Lean kernel only verifies what is explicitly linked.
- You may prove as many helper lemmas as you want in separate files.
- However, the formalization is only considered "complete" when the top-level Anchor theorem compiles with zero `sorry`s, proving that all the helper lemmas successfully link together under the Lean kernel's strict type checking.

*Remember: Lean is an end-to-end verifier, not a spell-checker for isolated logic snippets.*
